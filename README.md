# Лабораторна робота №1

**Дисципліна:** Тестування програмного забезпечення  
**Тема:** Розробка та тестування програми для аналізу взаємного розміщення прямих на площині  
**Варіант:** 19 (1, 4, 6)  
**Мова програмування:** Java  
**Середовище розробки:** IntelliJ IDEA  
**Система збирання:** Maven

---

## 1. Постановка задачі

Розробити консольну програму, яка:

- приймає на вхід три прямі, задані різними способами (відповідно до варіанту);
- приводить кожну пряму до загального вигляду `Ax + By + C = 0`;
- визначає взаємне розміщення трьох прямих;
- знаходить точки перетину (якщо вони існують);
- коректно обробляє помилкові та некоректні вхідні дані;
- проходить тестування за методом класів еквівалентності та граничних значень.

Програма повинна вивести **одне** з наступних повідомлень:

- «Прямі співпадають»
- «Прямі не перетинаються»
- «Єдина точка перетину прямих (x0, y0)»
- «Дві точки перетину прямих (x1, y1), (x2, y2)»
- «Три точки перетину прямих (x1, y1), (x2, y2), (x3, y3)»

## 2. Вхідні дані (варіант 19: 1, 4, 6)

### Пряма 1 — загальний вигляд (формула 1)

**Рівняння:**
`a1·x + b1·y + c1 = 0`

**Вхідні дані:**  
`a1, b1, c1`

**Умова коректності:**  
`a1² + b1² ≠ 0`


---

### Пряма 2 — у відрізках (формула 4)

**Рівняння:**
`x / a2 + y / b2 = 1`


**Вхідні дані:**  
`a2, b2`

**Умова коректності:**  
`a2 ≠ 0, b2 ≠ 0`


---

### Пряма 3 — через точку і нормаль (формула 6)

**Рівняння:**
`a3(x − x0) + b3(y − y0) = 0`

**Вхідні дані:**  
`x0, y0, a3, b3`

**Умова коректності:**  
`a3² + b3² ≠ 0`

---

### Обмеження на значення

Усі вхідні параметри — **цілі числа** в діапазоні:
[-119; 119]

## 3. Математичні формули

### 3.1. Загальний вигляд прямої

Усі прямі приводяться до вигляду:
Ax + By + C = 0

---

### 3.2. Приведення до загального вигляду

#### Загальний вигляд (формула 1)
```
A = a1
B = b1
C = c1
```

#### У відрізках (формула 4)
```
x / a2 + y / b2 = 1
b2·x + a2·y − a2·b2 = 0
A = b2
B = a2
C = −a2·b2
```

#### Через точку і нормаль (формула 6)
```
a3(x − x0) + b3(y − y0) = 0
a3·x + b3·y − (a3·x0 + b3·y0) = 0
A = a3
B = b3
C = −(a3·x0 + b3·y0)
```

---

## 4. Взаємне розміщення прямих

### 4.1. Визначник системи

Для двох прямих:
`A1·x + B1·y + C1 = 0`
`A2·x + B2·y + C2 = 0`

Визначник:
`D = A1·B2 − A2·B1`

---

### 4.2. Умови

- `D = 0` і коефіцієнти пропорційні → прямі співпадають
- `D = 0` і коефіцієнти не пропорційні → прямі паралельні
- `D ≠ 0` → прямі перетинаються

---

### 4.3. Формули точки перетину (метод Крамера)
```
x = (B1·C2 − B2·C1) / D
y = (C1·A2 − C2·A1) / D
```
---

### 4.4. Порівняння з нулем

Для дійсних чисел використовується:
```
ε = 10⁻⁸
|z| ≤ ε → z = 0
```


---

## 5. Класи еквівалентності

### 5.1. Припустимі класи

- **P1** — прямі співпадають
- **P2** — прямі не перетинаються
- **P3** — єдина точка перетину
- **P4** — дві точки перетину
- **P5** — три точки перетину

---

### 5.2. Неприпустимі класи

- **N1** — неціле значення
- **N2** — значення поза допустимим діапазоном
- **N3** — `a1 = 0` і `b1 = 0`
- **N4** — `a2 = 0` або `b2 = 0`
- **N5** — `a3 = 0` і `b3 = 0`

## 7. Тестування

Тестування виконано:

- методом **класів еквівалентності**;
- методом **граничних значень**;
- автоматичними **юніт-тестами (JUnit 5)**;
- інтеграційними тестами консольного застосунку.

Для кожного припустимого класу еквівалентності реалізовано **не менше 6 тестів**,  
для кожного неприпустимого класу — **не менше 3 тестів**.

Усі автоматичні тести пройдені успішно.
